## 一、产品定位与商业模式

### 1.1 核心定位

Token Plan 团队版（企业版）是阿里云百炼面向企业用户推出的 AI 模型订阅服务，与面向个人用户的个人版并行存在。其商业模式核心是：

- **预付制订阅** — 按月/按年购买坐席，绑定固定 Credits 配额
- **Credits 作为统一计量货币** — 屏蔽不同模型间的价格差异，企业只需关注 Credits 消耗
- **弹性补充机制** — 通过共享用量包应对突发流量，不强制升档

### 1.2 Credits 定价体系 ✅ 高可信

**数据来源**：官方帮助文档 + 官方开发者社区 + 多个第三方博客交叉印证，数字数据完全一致。

#### 坐席定价（月付）

| 坐席档次 | 月费 | Credits 配额/月 | 单价（元/Credit） | 适用人群参考 |
|---------|------|--------------|----------------|-----------|
| 标准坐席 | ¥198 | 25,000 | 0.00792 | 轻度使用，产品/测试类 |
| 高级坐席 | ¥698 | 100,000 | 0.00698 | 日常高频，工程师/分析师 |
| 尊享坐席 | ¥1,398 | 250,000 | 0.005592 | 重度依赖，AI工程师/平台开发 |

> **注意**：坐席名称在不同来源中存在差异（旗舰版/尊享版），官方正式名称为「标准坐席」「高级坐席」「尊享坐席」，数字数据无争议。

**档位价格梯度**：
- 高级坐席 vs 标准坐席：用量 4× 但价格仅 3.5×，性价比更优
- 尊享坐席 vs 标准坐席：用量 10× 但价格仅 7×，大量使用时优先升档而非买用量包

#### 共享弹性用量包

| 规格 | 单价 | Credits | 单价（元/Credit） | 有效期 |
|------|------|---------|----------------|------|
| 标准包 | ¥5,000 | 625,000 | **0.008** | 购买后 1 个月，到期清零 |

> ⚠️ **重要警示**：共享用量包单价（0.008元/Credit）**高于**尊享坐席单价（0.005592元/Credit），大量使用时应优先升级坐席档位而非依赖用量包补充。

## 二、坐席管理机制

### 2.1 核心规则 ✅ 高可信

**数据来源**：官方帮助文档 `help.aliyun.com/zh/model-studio/token-plan-team` 及 FAQ

| 规则维度 | 具体内容 |
|---------|---------|
| 一人一席 | 每个成员同一时间只能持有一个席位，席位不可跨成员共享 |
| API Key 管控 | 成员各自使用独立 API Key，但**成员无法自行生成**，须联系管理员获取 |
| 席位到期 | 席位到期后对应 API Key **立即失效**，需重新订阅并分配席位才能恢复 |
| 弹性补购 | 新增席位与现有订阅**统一到期**，费用按剩余时长折算 |

**弹性补购示例**：订阅周期剩余 15 天时加购标准坐席，仅收取 15 天费用（¥198 × 15/30 = ¥99）。

### 2.2 权限层级

```
管理员（Admin）
  ├── 统一分配 API Key 给团队成员
  ├── 管理席位购买、升降档、到期续订
  ├── 配置共享用量包
  └── 分配账户配额至多个业务空间（Workspace）

团队成员（Member）
  ├── 使用管理员分配的独立 API Key
  ├── 消耗本坐席月度 Credits + 共享用量包
  └── ❌ 无法自行生成 API Key
      ❌ 无法查看或管理其他成员的用量
```

### 2.3 多业务空间（Workspace）隔离 ✅ 高可信

**数据来源**：官方帮助文档 `help.aliyun.com/zh/model-studio/use-workspace`

账户总配额可按比例分配给多个业务空间，实现多项目隔离管理：

```
账户总配额：1000 QPM
  ├── prod-workspace：600 QPM（生产环境）
  ├── test-workspace：200 QPM（测试环境）
  └── dev-workspace：100 QPM（开发环境）
```

**企业价值**：不同项目、不同团队的用量互不干扰，避免单一项目流量突增影响全局。

### 2.4 尚待明确的问题

- 坐席总数量是否有上限？大型企业（100 席以上）是否有专属折扣或框架协议？
- 是否支持对公转账、合同签订、发票开具等企业采购合规流程？

## 三、配额抵扣规则

### 3.1 三级抵扣优先级 ✅ 高可信

**数据来源**：官方帮助文档明确记载，多源一致，无歧义。

```
第一优先：坐席月度额度
      ↓ 耗尽后
第二优先：共享用量包（多个用量包时，按最近到期日优先消耗）
      ↓ 全部耗尽后
第三级：服务直接暂停，等待下一计费周期或手动购买共享用量包
```

> ❌ **已被验证为不准确**：「额度耗尽后自动转按量付费」——此说法 0/3 票通过验证。官方明确：**无按量付费兜底机制**，额度耗尽后服务直接暂停。

### 3.2 Credits 到期清零规则 ✅ 高可信

| 类型 | 有效期 | 到期处理 | 是否结转 |
|------|------|---------|---------|
| 坐席月度 Credits | 当月有效 | 自动清零 | ❌ 不结转 |
| 共享用量包 Credits | 购买后 1 个月 | 自动清零 | ❌ 不结转 |
| 节省计划（Savings Plan） | 月承诺周期 | 自动过期 | ❌ 不可累积 |

> ⚠️ **高风险点**：三种类型的 Credits 均不结转，月末剩余即损失。需建立用量预测机制，避免浪费。

### 3.3 节省计划（Savings Plan）与 Token Plan 的抵扣优先级

节省计划是与 Token Plan 并行存在的另一套体系，其内部抵扣顺序为：

```
免费额度 → 资源包 → 其他模型节省计划 → AI 通用型节省计划 → 按量付费
（同类计划中，先消耗先到期的）
```

> ⚠️ 两套体系适用场景不同，混用时需注意抵扣顺序的相互影响。

## 四、企业版 vs 个人版完整对比

### 4.1 核心差异对比表 ✅ 高可信（企业版数据）/ ⚠️ 中可信（个人版限制细节）

| 维度 | 个人版 | 企业版（团队版） |
|------|------|--------------|
| **用户规模** | 单一用户 | 多席位，支持团队协作 |
| **席位管理** | 无 | 管理员统一管控，一人一席 |
| **API Key** | 用户自行生成 | 管理员分配，成员无法自行生成 |
| **Credits 配额** | 个人档位限制 | 三档坐席 + 共享用量包弹性叠加 |
| **业务空间隔离** | ❌ 不支持 | ✅ 支持多 Workspace 按比例分配配额 |
| **企业数据隐私** | 标准保障 | 企业级数据隐私保障 |
| **成本归因** | 个人维度 | 可按坐席/Workspace 追踪成本 |
| **配额共享** | 无 | 共享用量包可跨坐席使用 |
| **适用场景** | 个人开发、学习、小型项目 | 团队协作、企业项目、多业务线管理 |

### 4.2 选择决策树

```
团队人数 ≤ 1 人，且无多项目隔离需求？
  └── 是 → 个人版（更灵活，无席位管控开销）
  └── 否 ↓

需要 API Key 统一管控 / 成本归因到个人？
  └── 是 → 企业版（坐席管理机制天然支持）
  └── 否 ↓

需要多项目/多部门配额隔离？
  └── 是 → 企业版（Workspace 机制）
  └── 否 → 视用量规模选择个人版或企业版
```

## 五、实际使用注意事项与风险

### 5.1 高风险点清单

| 风险 | 描述 | 应对措施 |
|------|------|---------|
| 🔴 服务中断 | 额度耗尽后**无兜底**，直接暂停服务 | 建立月度用量预警（建议 70% 时触发告警） |
| 🔴 弹性包浪费 | 共享用量包到期清零，且单价高于尊享坐席 | 预测准确时优先升档，峰值突发再购弹性包 |
| 🟡 API Key 失效 | 席位到期后 API Key 立即失效 | 提前 7 天续订，避免生产环境中断 |
| 🟡 Credits 不结转 | 月末剩余 Credits 全部清零 | 月末低峰期可主动提交批量任务消耗配额 |
| 🟡 弹性包单价倒挂 | 大量购买弹性包成本高于升档 | 月用量超过当前档位 120% 时考虑升档 |

### 5.2 成本优化最佳实践

**短期可立即实施**

1. **启用模型缓存**：系统 Prompt 命中缓存时 Credits 消耗大幅降低（Qwen3.6-Plus 缓存 Token 约为正常输入的 49%）
2. **按档位匹配用量**：月用量稳定在 8 万 Credits 以上时，高级坐席（10 万 Credits/¥698）比标准坐席（2.5 万 Credits/¥198 × 4 = ¥792）节省 ¥94/月
3. **错峰批量任务**：在月初配额充足时集中执行高耗量任务，避免月末配额不足

**架构层面**

4. **Workspace 分层管理**：生产/测试/开发环境分配不同 Workspace，防止测试流量抢占生产配额
5. **应用层用量埋点**：在 API 调用层统计每个业务线的 Credits 消耗，辅助下月采购决策
6. **Prompt 精简**：去除冗余描述，RAG 场景控制 Context 长度，可降低 30–50% 输入 Credits

### 5.3 采购决策参考

**按月人均用量的档位选择建议**

| 月人均 AI 调用量（估算） | 推荐档位 | 理由 |
|----------------------|---------|------|
| < 2 万 Credits/人/月 | 标准坐席 ¥198 | 用不满高级坐席，升档浪费 |
| 2–8 万 Credits/人/月 | 标准坐席 × N | 按需购买，弹性包补充峰值 |
| 8–20 万 Credits/人/月 | 高级坐席 ¥698 | 性价比优于标准 × 4 |
| > 20 万 Credits/人/月 | 尊享坐席 ¥1,398 | 单价最低（0.005592元/Credit） |
