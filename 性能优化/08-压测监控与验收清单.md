# 压测、监控与验收清单

## 1. 压测矩阵

| 原始 QPS | 每窗口聚合键 | 目的 |
|---:|---:|---|
| 5,000 | 当前规模 | 建立基线 |
| 20,000 | 10,000 | 验证线性增长 |
| 50,000 | 10,000 | 高聚合比场景 |
| 50,000 | 50,000 | 中期目标 |
| 50,000 | 100,000 | 规划上限 |
| 75,000 | 100,000 | 突发安全余量 |
| 50,000 | 100,000 + 热点 Key | 流量倾斜 |

分别压测 4、8、12/16 分区候选，但仅在前一档不能达标时测试下一档。

## 2. 故障注入

- 数据库事务提交前强杀 Pod；
- 数据库提交后、Kafka Offset 提交前强杀；
- Rebalance 时旧、新 Pod 重叠；
- 单个 Pod 长时间 GC 或网络隔离；
- 两个 Pod 同时固化一个窗口桶；
- 聚合事务与窗口固化并发；
- 两个 Worker 同时领取同一 Outbox；
- Kafka 发送成功、更新 `SENT` 前崩溃；
- 数据库、Redis、元数据服务分别不可用 5～10 分钟；
- 重复 `messageId`、重复 Offset、乱序和迟到消息；
- MySQL、PostgreSQL 分别执行完整测试。

## 3. Kafka 指标

- 每分区输入和消费速率；
- Consumer Lag、增长速度、预计追平时间；
- Rebalance次数、耗时和移动分区数；
- Kafka Offset与DB Checkpoint差值；
- Pause/Resume次数；
- 最热分区与平均分区流量比。

## 4. 聚合服务指标

- 每窗口唯一键数；
- 原始消息/聚合键比；
- 本地状态条目数与字节数；
- 批量大小和刷新耗时；
- Offset缺口、倒退和重叠次数；
- Heap、GC、CPU和锁竞争；
- 迟到与修正事件数。

## 5. 数据库指标

- Upsert P50/P95/P99；
- 事务持续时间和回滚率；
- Checkpoint行锁等待；
- 行锁、死锁和重试；
- WAL/Redo速率、IOPS和磁盘延迟；
- 每日分区行数、数据和索引大小；
- Outbox领取与状态更新耗时。

## 6. 话单指标

- 待固化窗口桶数量；
- 窗口自然结束到固化完成的延迟；
- Outbox积压和最老任务年龄；
- 话单构建/发送吞吐；
- 元数据缓存命中率和批量查询耗时；
- Kafka发送失败、重试和重复发送；
- 原始用量、聚合事实、话单三层对账差异。

## 7. 验收门槛

- 50k QPS 持续运行时 Lag 不发散；
- 75k QPS突发后可在约定时间内追平；
- 单 Pod 故障时服务继续消费；
- 10万聚合键在窗口结束后5分钟内完成上报；
- 相同 Offset重放后Token不增加；
- 数据库提交前崩溃后不漏计；
- 相同 `bill_id` 重发只结算一次；
- 元数据查询量与唯一ID数相关，而不是与原始QPS相关；
- MySQL、PostgreSQL结果语义一致；
- 所有正式切换都有经过验证的回滚开关。

## 8. 推荐验证命令

应先检查工具是否存在，再按仓库实际工程执行：

```bash
command -v go
go test ./...
go test -race ./...
go vet ./...
```

Kafka、MySQL和PostgreSQL集成测试必须连接隔离环境和测试 Topic，不连接生产结算 Topic，不占用生产后端 `127.0.0.1:8080`。
